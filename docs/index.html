Design Philosophy

Purpose-Built for Healthcare QA

⚡

Shift-Left Testing

RestAssured API calls seed patient records and appointments, then Selenium validates the UI layer. 10–100× faster than UI-based setup with zero fragility.

🔒

RBAC Security Testing

Data-driven parameterization across Admin, Doctor, Billing Clerk, Intern, and Front Desk roles. Every permission validated against a centralized roles.json matrix.

🌐

BiDi Network Interception

Selenium 4 Bidirectional Protocol monitors all network requests during execution. Catches silent API failures — critical in healthcare where unlogged data loss violates HIPAA.

💥

Chaos Engineering

BiDiChaosTest injects network failures during active sessions to verify graceful error handling, zero silent data loss, and clean JavaScript console output.

Getting Started

Up and Running in Minutes

Prerequisites: Java 21 (Eclipse Temurin), Maven 3.9+, Chrome (latest stable)

# Full suite (visible browser)
mvn clean test

# CI mode (headless)
mvn clean test -Dheadless=true

# Specific suite only
mvn clean test -Dtest=RBACSecurityTest

# Override browser
mvn clean test -Dbrowser=firefox

Generate rich Allure Reports with test timelines, screenshots on failure, and step-by-step traceability.

# Open report in browser
mvn allure:serve

# Generate to target/site/
mvn allure:report

GitHub Actions pipeline runs automatically on every push to main/develop:

# Pipeline stages:
1. Provision — JDK 21 + Chrome
2. Execute  — Full test suite in headless Chrome
3. Report  — Allure report generated & uploaded
4. Deploy  — Report published to GitHub Pages
Engineering

Key Design Decisions

ThreadLocal<WebDriver>
Thread-safe parallel execution without test interference
BiDi over CDP
Future-proof W3C standard vs. deprecated Chrome DevTools Protocol
RestAssured for test data
10–100× faster than UI-based setup; decouples data seeding from UI state
JSON data files
Readable by QA, editable without code changes, version-controlled
SoftAssert in RBAC
Captures ALL permission violations per role vs. failing on first
Mock webapp included
Framework is fully self-contained; no external dependencies needed
